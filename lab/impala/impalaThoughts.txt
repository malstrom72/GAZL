Multiple assignments
====================

With strong typechecking on function calls (if we would at least require number of args and returns for function
pointers etc) we could use multiple returns on "inner function calls", e.g.:

// now if f2 returns 3 results, f1 will receive those between c and g, f1 returns 2 vars and we add a last one
a,b,c=f1(a, b, c, f2(d, e, f), g, h),last_one

// we have another syntax for extracting a certain result
a=f1(a,b,c,f2(d,e,f):2,g,h):1

The implementation would work like this: return values are placed before arguments on the stack and we always build
expression lists on the stack. In the example above, a,b,c would be placed in %2,%3,%4 (return of f1 is %0,%1), f2
is called with %5 as offset, and it's argument would be placed in %8,%9,%10.

Short-circuit evaluation
========================

Each element can be one of:

	*) branch on false (F->FALSE x:, created by AND)
	*) branch on true (T->TRUE x:, created by OR)
	*) invert (F<=>T, created by NOT)
	*) (unique) false branch label (FALSE x:)
	*) (unique) true branch label (TRUE x:)
	*) comparison operation (expr == expr, expr < expr etc etc)
	*) any other instruction (effective, e.g. adding numbers or ineffective, e.g. a nop)

There can only be one branch on false / true per comparison, and there can't be no other effective instructions
in between (but there can and will be labels).

Process bottom to top:

	*)	branch on false or true :
		assign the false branch target or true branch target (use the alias map if there is an entry for this target)
		set current branch condition to true or false
		remove
		
	*)	not :
		swap the false and true branch targets
		invert current branch condition
		remove
	
	*)	false or true branch target :
		if there is an assigned branch target for this condition: map an alias of this label to it and remove
		otherwise assign the false branch target or true branch target (leave the label)

	*)	comparison operation :
		if current branch condition is false, invert the operation (e.g. "==" -> "!=" , ">" -> "<=" etc)
		(if current branch condition is not false or true this indicates a programming error)
		assign the target label for the operation to the branch target of the current branch condition
		clear the branch target of both branch conditions
		
	*)	any other effective instruction :
		clear the branch target of both branch conditions
